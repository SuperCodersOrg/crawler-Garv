### Session 1
* Date: 15-07-2026 (10.00AM-1.30PM)

* Duration: 3.5 hours

* Goal: Resolve Chrome process startup failures, ensure headless Chrome remains completely invisible on Windows, and fix socket port binding issues.

* Problem Encountered: 
  - Chrome was opening a visible, unresponsive, and greyed-out platform window despite being started with headless parameters.
  - Zombie `chrome.exe` processes from previous interrupted runs stayed active in background, holding port 9222 and causing new connection attempts to fail with socket bind errors.
  - CDP initialization would fail immediately if Chrome took more than a split-second to spin up.

* Outcome: 
  - Integrated Windows Job Objects API (`CreateJobObject`, `SetInformationJobObject`, `AssignProcessToJobObject`) with Chrome child process control in `ChromeProcess.cpp` to ensure automatic process cleanup when the crawler exits.
  - Appended `--window-position=-2400,-2400` to Chrome process command flags, pushing the browser window far off-screen to guarantee invisibility on Windows.
  - Added configurable retry loops (`connectRetries_` and `connectDelayMs_`) in `BrowserRenderer::initializeCDP()` to make connection handshakes tolerant of slow Chrome spin-up times.
