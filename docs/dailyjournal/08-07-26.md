## Section 2 – Failed Attempt

At the start I spent quite some time deciding whether I should build the parser using `libcurl`, regular expressions or write it completely myself.

Using `libcurl` would've solved most of the parsing work, but I wanted to understand how URL parsing actually works internally and also keep this part of the crawler independent. Regex also looked tempting at first, but it would've become messy for handling all the URL edge cases.

I finally decided to write my own parser using `std::string_view` and a simple state-based approach. The first version only worked for very basic URLs and almost every test failed because I wasn't extracting the scheme, host and path correctly. After fixing those, I realised query strings were still being parsed incorrectly, so I extended the parser to properly separate paths and query parameters.

Using `string_view` also helped avoid creating unnecessary temporary strings while scanning the URL, which kept the implementation simpler and linear.

## Section 4 – Learning Reflection

Today I understood why parsing URLs manually is much harder than it looks. At first it seems like splitting a string at `://` and `/` should be enough, but once queries, ports, credentials and different URL formats come into the picture, there are many edge cases that need to be handled carefully.

I also realised that writing unit tests alongside the implementation makes debugging much easier. Instead of manually printing values every time, I could immediately see which part of the URL wasn't being parsed correctly. The first version failed almost every test, but after fixing one issue at a time I could clearly see the progress until every test passed.

Another thing I learnt was that `std::string_view` fits this kind of parser really well because most of the work only involves looking at different parts of the original string instead of creating new strings repeatedly. It made the parser cleaner and also avoided unnecessary copying.

By the end of the day I had a working custom `URLParser` that could parse schemes, hosts, ports, paths, credentials and query parameters without relying on external libraries.
