## Section 2 – Failed Attempt

While implementing the blacklist and link filter, I first extracted the file extension directly from the end of the URL. That worked for simple URLs but immediately failed for links containing query strings or fragments like `.css?v=2` or `.png#icon`. Because of that, valid filtering logic started giving incorrect results.

Instead of checking from the front, I changed the approach and scanned backwards to locate `?` and `#` first. That allowed me to isolate the actual path before extracting the extension, which fixed the remaining edge cases.

For the CDP integration, I initially assumed the browser process itself was failing, but Chrome was actually launching correctly. The real problem was how it was being launched and attached inside the crawler project. After updating the integration and fixing a few configuration issues, the browser started working properly with the crawler.

## Section 4 – Learning Reflection

Today was mostly about connecting different crawler components instead of building completely new ones.

While working on the link filter, I realised that URLs often contain much more than just a file path. Query strings and fragments can completely change how a URL looks even though the actual resource stays the same. Parsing those correctly before checking extensions made the filtering logic much more reliable.

Building the `SeenStore` also made me appreciate why I had spent time implementing my own HashMap earlier. Instead of creating another data structure, I could directly reuse it for tracking visited URLs, which made the implementation straightforward and kept the overall design modular.

Integrating the CDP module into the actual crawler project also showed me that getting independent components to work together is often harder than implementing them individually. The browser automation code had already worked in isolation, but integrating it into the crawler introduced configuration and profile issues that only appeared when everything was combined.

By the end of the day I had implemented the blacklist and link filtering system with comprehensive unit tests, added the `SeenStore` using my HashMap to prevent duplicate crawling, fixed the remaining edge cases in URL filtering, and successfully integrated the CDP browser module into the crawler project so it could be used by the rest of the crawler pipeline.
