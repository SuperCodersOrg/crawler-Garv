## Section 2 – Failed Attempt

Initially I thought most of the crawler was finished, so I started doing full crawl tests instead of testing individual modules. Those runs exposed a few small logic bugs that hadn't shown up in unit tests because they only appeared when all the components worked together.

While testing headless mode, I first assumed Chrome itself had a problem with rendering pages. After trying different launch options, I realised the issue wasn't with rendering but with how the browser window was being created. I fixed it by moving the browser window off-screen instead of relying entirely on the previous headless setup, which made the crawler behave much more consistently during long runs.

I also noticed that changing database settings required editing source code every time. To avoid rebuilding the project for simple configuration changes, I introduced a configuration file that stores database credentials and crawler parameters, along with a parser that loads these values when the crawler starts.

## Section 4 – Learning Reflection

Today made me realise that once the main implementation is complete, most of the remaining work shifts from writing new features to improving reliability.

Testing the crawler from start to finish exposed issues that individual unit tests couldn't catch. Many of the bugs were small on their own, but together they affected how smoothly the crawler behaved during longer crawl sessions.

Adding a configuration file also made the project much easier to maintain. Earlier, changing database credentials or crawler settings meant modifying source code and rebuilding the project. Separating configuration from implementation makes future testing much faster and keeps the crawler more flexible.

The resume functionality was another useful improvement. Instead of always starting from the seed URL, the crawler can now continue from where it stopped after an interruption. That makes the crawler much more practical for larger crawling tasks where stopping and restarting is expected.
