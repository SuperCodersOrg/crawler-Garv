# Daily Design Journal · Day 13

## Section 1 – Specific Bug

Today's focus was resolving interface integration defects and securing headless browser stability during background automated execution.

I encountered a crash in `QuickFilter::shouldparse` where whitespace-trimming empty inputs (for example, pure space inputs) caused the crawler to execute undefined memory actions on `url.front()`. Additionally, headless execution launched visible Chrome windows that were greyed out and unresponsive, lock-binding TCP port 9222 and failing to close, which blocked subsequent crawl initializations.

---

## Section 2 – Failed Attempt

Initially, I tried to handle Chrome's window unresponsiveness by adjusting basic Chrome CLI viewport parameters, hoping Chrome would boot fully headless. However, standard browser process wrappers failed to completely hide the frame on Windows. I addressed this by offsetting the platform coordinates deep off-screen.

To prevent port bind conflicts, I initially tried a simple short connection sleep. This did not solve the issue when zombie browser tasks were left running from crashed crawls. I resolved this by integrating Windows Job Objects to force-terminate all child Chrome process trees when the C++ host crawler exits. Finally, I added a safety sentinel check inside `QuickFilter` to exit early on empty URL buffers.

## Section 4 – Learning Reflection

Today emphasized that stability and clean system teardowns are as critical as core feature development in production-grade systems.

Individual unit tests missed port-lock boundaries because they assume a fresh browser startup. Designing process management using OS Job Objects ensures that child windows are swept clean, avoiding resource leaks. Furthermore, simple boundary errors—like dereferencing an empty trimmed string—highlight the necessity of defensive input-handling guard screens across all filter pipelines.

By the end of the day, browser processes were successfully hidden off-screen, zombie port-lock issues were resolved via child process grouping, filter bounds checked correctly, and the Version 4 design documentation was finalized.
